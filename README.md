# Push Express

Push Express tracks the latest Canadian Express Entry draw and sends web push notifications to subscribed visitors when a new draw is detected.

The backend is built with FastAPI, SQLAlchemy, and `pywebpush`. It periodically fetches the IRCC Express Entry rounds JSON feed, stores the latest response in a local backup file, and notifies saved browser push subscriptions when the draw data changes.

## Features

- Fetches the latest Express Entry draw from the official IRCC JSON feed.
- Normalizes draw names into short program labels such as `CEC`, `PNP`, `GEN`, and `STEM`.
- Exposes the latest draw summary through a FastAPI endpoint.
- Stores browser push subscriptions in a SQLAlchemy database.
- Sends push notifications using VAPID keys.
- Includes a small sample frontend for displaying the latest score and subscribing to notifications.

## Requirements

- Python 3.10+
- `pip`
- A database URL supported by the current SQLAlchemy configuration
- A browser that supports service workers and push notifications

> Note: `database.py` currently passes `connect_args={"check_same_thread": False}` to SQLAlchemy, which is intended for SQLite. Use a SQLite URL unless you update the database engine configuration for another database.

## Setup

Run the setup script from the project root:

```bash
source setup.sh
```

The script will:

- Create a virtual environment named `.push-express` if one does not already exist.
- Activate the virtual environment.
- Install dependencies from `requirements.txt`.
- Generate VAPID keys.
- Create a `.env` file containing `PRIVATE_KEY` and `PUBLIC_KEY` if `.env` does not already exist.

After the script finishes, add the remaining required values to `.env`.

```env
DATABASE_URL=sqlite:///./push_express.db
EMAIL=mailto:you@example.com
TIMER=300
```

### Environment Variables

| Variable | Required | Description |
| --- | --- | --- |
| `PRIVATE_KEY` | Yes | VAPID private key generated by `setup.sh`. |
| `PUBLIC_KEY` | Yes | VAPID public key sent to the browser for push subscriptions. |
| `DATABASE_URL` | Yes | SQLAlchemy database URL. SQLite is currently the safest default. |
| `EMAIL` | Yes | VAPID subject claim. Use a `mailto:` URL such as `mailto:you@example.com`. |
| `TIMER` | No | Draw polling interval in seconds. Defaults to `300`. |
